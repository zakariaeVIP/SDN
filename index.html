Introduction aux réseaux modernes

1 Infrastructure traditionnelle

L'infrastructure réseau traditionnelle présente plusieurs inconvénients :

  • Coûts d'équipement élevés
  • Limitations matérielles
  • Administration complexe
  • Problèmes de compatibilité

2 Évolution des besoins

Les besoins des entreprises modernes nécessitent des réseaux :

  • Plus flexibles et évolutifs
  • Facilement administrables
  • Économiques
  • Tolérants aux pannes

Éléments de sécurité réseau

Pare-feu (Firewall)

Rôle principal : Filtrage

Types :

  • Fortinet
  • Palo Alto
  • PF Sense
  • ASA

IDS/IPS

IDS : Intrusion Detection System

IPS : Intrusion Prevention System

Ces systèmes permettent de détecter et prévenir les tentatives d'intrusion sur le réseau.

Autres éléments

  • VPN : Connexions sécurisées à distance
  • Proxy : Filtrage et contrôle d'accès
  • Zone DMZ : Zone démilitarisée entre réseau interne et externe
  • Protocoles & Ports : HTTP (80), HTTPS (443), etc.

Virtualisation

La virtualisation permet d'implémenter plusieurs machines virtuelles sur un seul serveur physique.

Infrastructure traditionnelle

Problèmes :

  • Gaspillage de performances
  • Coût élevé (un service = une machine)
  • Administration complexe
  • Faible évolutivité

Solution : Virtualisation

Types de virtualisation :

  • Type 1 (natif / bare metal) : Plus rapide, fonctionne directement sur le matériel (ex: Hyper-V)
  • Type 2 (hébergé) : Fonctionne sur un système d'exploitation hôte (ex: VirtualBox, VMware Workstation)

Cloud Computing

Accès universel

Accès à tout moment et de n'importe quel lieu

Scalabilité

Évolutivité automatique selon les besoins

Réduction des coûts

Paiement à l'usage, sans investissement lourd

Compatibilité

Interopérabilité entre différents services

Modèles de Cloud ▼
  • Cloud Public : Accessible à tous (Microsoft, Amazon, Google)
  • Cloud Privé : Infrastructure locale et personnelle
  • Cloud Hybride : Combinaison du public et du privé
  • Cloud Communautaire : Partagé entre plusieurs entreprises
Services Cloud ▼
  • IaaS (Infrastructure as a Service) : Infrastructure réseau en mode cloud (ex: Microsoft Azure, Amazon EC2)
  • PaaS (Platform as a Service) : Environnement de développement
  • SaaS (Software as a Service) : Service applicatif distant (ex: Google Docs, Office 365)

Software Defined Networking (SDN)

Réseau traditionnel

Inconvénients :

  • Administration complexe (configuration manuelle)
  • Évolutivité difficile
  • Tolérance aux pannes limitée
  • Gaspillage de temps
  • Problèmes de compatibilité

Architecture SDN

Avantages :

  • Administration centralisée et simplifiée
  • Évolutivité simple et rapide
  • Meilleure tolérance aux pannes
  • Gain de temps considérable
  • Standardisation (indépendance des constructeurs)

Comparaison des plans dans les réseaux

Réseaux traditionnels

Tous les plans sont intégrés dans chaque équipement :

  1. Plan de gestion (Commandes)
  2. Plan de contrôle
  3. Plan de données (CPU + RAM)

Problèmes : Charge sur CPU, latence, difficulté d'optimisation

Réseaux SDN

Séparation des plans :

  1. Plan de gestion → Applications
  2. Plan de contrôle → Contrôleur SDN centralisé
  3. Plan de données → Équipements réseau

Avantage : Gestion centralisée et intelligente

Architecture SDN

1. Couche Application

Applications métiers, scripts (Terraform, Ansible), APIs (REST)

2. Couche de Contrôle

Contrôleur SDN (APIC, ONOS, OpenDaylight)

3. Couche Infrastructure

Équipements physiques (switches, routeurs) avec OpenFlow

Types de fonctionnement OpenFlow

Réactif

Les paquets inconnus sont envoyés au contrôleur qui ajoute des règles à la volée.

Avantage : Dynamique

Inconvénient : Plus lent (latence supplémentaire)

Proactif

Les règles sont chargées à l'avance dans la table OpenFlow dès le démarrage.

Avantage : Rapide, faible latence

Inconvénient : Moins flexible (règles fixes)

Types de paquets en SDN (OpenFlow) ▼

Paquets Asynchrones

  • Echo / Request : Découverte des contrôleurs SDN
  • Port Status : Information sur les modifications d'une entrée, gestion des erreurs

Packet Out

  • Permet d'ajouter, supprimer ou modifier une règle dans la table OpenFlow
  • Utilisé par le contrôleur vers les équipements réseau
Infrastructure as Code (IaC) ▼

L'Infrastructure as Code permet le déploiement automatique de l'infrastructure réseau via des scripts et des langages de programmation.

  • Automatisation complète de l'infrastructure
  • Utilisation d'outils comme Terraform ou Ansible
  • Redondance et résilience configurées par code
  • Équilibrage de charge intelligent
Problématique d'équilibrage de charge ▼

Exemple : Deux serveurs web avec une charge inégale

Objectif :

  • Répartir les charges de façon uniforme
  • Éviter les goulets d'étranglement

Solution : SDN permet une gestion intelligente de l'équilibrage de charge via des règles programmables
